# ドメインモデルファースト開発

テストファーストでドメインモデルから実装を開始し、インフラストラクチャの詳細を後回しにする開発手順。

## 開発順序

1. **ドメインモデルのテストと実装** - インフラに依存しないビジネスルールのテスト
2. **インメモリリポジトリ** - テスト用のシンプルなリポジトリ実装
3. **ユースケースのテストと実装** - ドメインモデルを使うアプリケーション層
4. **統合テスト** - システム全体の動作確認
5. **インフラストラクチャ実装** - DB、外部API等の実装

## 原則

- ドメインモデルはフレームワークやDBに依存しない
- テストはインメモリ実装で高速に回す
- インフラの詳細は最後に決定する
- 設計が歪むリスクを最小化する

## リスク（順序を守らない場合）

- 設計の歪み: インフラの制約がドメインモデルに漏れる
- 技術の漏洩: ORMやフレームワークの都合がモデルを汚す
- テスタビリティの低下: インフラ依存でテストが遅くなる
