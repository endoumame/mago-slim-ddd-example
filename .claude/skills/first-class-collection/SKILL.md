# First Class Collection

コレクションをラップするクラスは、コレクション以外のフィールドを持たない。

## 適用基準

- コレクションに対するビジネスロジックが複数箇所に散在
- ドメイン概念として自然に名前がつく（Orders, Inventory等）
- バリデーション・集計・フィルタリングのルールが存在
- 外部コードが直接コレクションを変更している

## 変換パターン

1. **集計**: sum/total/countをコレクションクラスに移動
2. **フィルタリング**: 散在するfilter式をドメインメソッドに
3. **バリデーション**: 構築時に制約を強制
4. **不変性**: 変更メソッドは新しいインスタンスを返す

## 関連スキル（併読推奨）
- `tell-dont-ask`: コレクションへのTDA適用
- `law-of-demeter`: コレクションへのLoD適用
- `domain-building-blocks`: 集約内のコレクション設計
