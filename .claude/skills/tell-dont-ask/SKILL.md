# Tell, Don't Ask

オブジェクトの内部状態を問い合わせて外部で判断せず、オブジェクトに直接命令する。

## 核心原則

getterで取得 → 外部で判定 → 操作 というパターンを避け、
オブジェクトに「やるべきこと」を直接命じる。

## 変換パターン

1. 条件ロジックをオブジェクトに移動
2. ポリモーフィズムで型分岐を置換
3. コレクション操作を包含オブジェクトに委譲
4. Null Objectパターンでnullチェックを排除
